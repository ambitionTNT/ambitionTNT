1、面向过程与面向对象有何区别？
   从语言方面出发：
	对于c语言来说，是完全面向过程的。
	对于c++语言来说，是一半面向过程，一半面向对象
	对于java语言来说，是完全面向对象的。

   什么是面向过程的开发方式？
      面向过程的开发方式主要的特点是：
	  注重步骤，注重的是实现这个功能的步骤。
		第一步干什么？
		第二步干什么？
		...
	  另外面向过程也注重功能的因果关系 。
	      因为 A 所以 B 
	      因为 C 所以 D
	      ...

	   面向过程中没有对象的概念。只要实现这个功能

	   面向过程的缺点？
		面向过程最主要是每一步与每一步的因果关系，其中A步骤因果关系到V步骤，A和B联合起来形成一个子模块，子模块和
		子模块之间有因为因果关系结合在一起，驾驶其中任何一个因果关系出现问题，此时整个系统的运转都会出现问题。
		（代码之间的耦合度太强，扩展力太差。）


	  面向过程的优点：
	     对于小型项目，采用面向过程的方式进行开发，效率过高，不需要进行对象的提取，模型的建立。
		快速开发。
	
  面向对象：
	采用面向对象的方式开发，更符合人类的思维方式。
	人类就是以“对象”的方式去认识世界的。
	所以面向对象更容易让我们接受欧

	面向对象就是将现实对象分割成不同的单元，然后对每个单元都实现成对象，然后驱动一下，让各个对象之间
	协作起来称为一个系统。

	采用面向对象的方式开发，耦合度低，扩展力强。
   

面向对象主要关注的是对象
面向过程主要关注的是过程

4、类和对象的概念：
 面向对象当中最主要的“一词”是：对象。
 
 什么是类？
	类实际上在现实世界中是不存在的，是一个抽象的概念，是一个模板。是我们人类大脑进行思考、总结、抽象的一个结果。
	（主要是因为人类的大脑不一般才有了类的概念）
	对象是通过类这个模板造出来的，类是一种抽象的类
	如动物、植物；猫科动物、犬科动物。吃肉动物、吃素动物；
	类本质上是现实世界中某些事物具有共同特征的集合。  
	对象 还有另一个名字叫 实例

	通过类这个模板创建对象的过程，叫做实例化。

	把对象特征提取出来，成为一个类的过程，叫做抽象。

	类 = 属性 + 方法；
	属性来源于：状态；
	方法来源于：动作；

	解决现实中的问题。也就是说 java软件能够模拟现实世界。
	java软件就是虚拟世界，用于模拟现实世界，解决现实问题。
	  
	  我们需要具备观察+抽象能力。我们需要观察现实世界当中对象1和对像2有什么共同特征，
	  然后在java程序中使用“类”来描述这些特征，然后通过 “类“ 实例化 ”对象“，让对象之
	  间互相协作起来形参系统，模拟现实世界。

9、什么是实例变量？
	对象又被称为实例。
	实例变量就是：对象级别的变量。

	如：
	public class 明星类{
	   double height;
	}

	身高这个属性所以的明星对象都有，但是每个明星都有自己的“身高”。
	假设创建10个对象，height变量应该是有10份。
	所以这种变量被称为对象级别的变量。属于实例变量。
	
	实例变量在访问的时候，是不是必须要创建对象？
		是的，不能通过类名.的方式直接访问成员变量
