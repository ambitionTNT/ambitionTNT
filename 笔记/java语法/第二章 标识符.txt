1.什么是命名规则？
			命名规则属于语法机制，必须遵守，不遵守命名规则标识不符合语法，
			这样，编译器会报错。

		规则1：标识符只能由数字、字母（包括中文）、下划线_、美元符号$组成，
		不能含有其它符号。

		规则2：标识符不能以数字开头

		规则3：关键字不能做标识符。例如：public class static void 这些蓝色的字体
		都是关键字，关键字是不能做标识符的。

		规则4：标识符是严格区分大小写的。大写A和小写a不一样。

		规则5：标识符理论上是没有长度限制的。
	尽管标识符区分大小写，但是对于类名来说，如果一个java源文件中同时出现了：A类和a类
	谁在前面，编译时就编译前面的，所以类名最好不要相同。
2.命名规范
   
   规范1：见名知意

   规范2：遵循驼峰命名方式

   规范3：类名、接口名有特殊要求
	类名和接口名首字母大写，后面每个单词首字母大写。
	StudenTest、UserTest,这是类名、接口名

   规范4：变量名、方法名有特殊要求
	变量名和方法名首字母小写，后面每个单词字母大写。
	name、studentName，这些是变量名、方法名。

   规范5：所以“常量”名全部大写。并且单词与单词之间采用下划线衔接。
	USER_AGR:用户年龄。
